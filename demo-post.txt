## 1. Demo 简介

**是什么：** 情绪团子是一个基于 Web 的 AI 情绪陪伴应用，打开浏览器就能用。它不是冰冷的记录工具，而是一个有 3D 形象、会说话、会回信的"情绪小精灵"，陪你画、听你说、读你写的、看你拍的，然后用 AI 真正"读懂"你的情绪并给出温暖回应。

**面向谁：** 
- 独居/独处的年轻人，想要一个随时能倾诉的对象
- 情绪容易起伏但不好意思跟朋友倒苦水的人
- 喜欢记录生活、做手帐、写日记的人

**主要功能（3 个核心）：**

1. **四种情绪表达 + AI 精准情绪识别**
   - 画一画（笔触分析）：在画布上涂鸦，系统通过笔触速度、压力、方差推断情绪
   - 说一说（语音分析）：对着麦克风说话，结合文本+声学特征识别情绪
   - 写一写（AI 情绪分析）：写下文字，调用大模型精准识别 6 种情绪（喜悦/悲伤/愤怒/焦虑/平静/无力），还能理解反讽和否定句
   - 拍一拍：用照片记录当下心情

   ![四种情绪表达入口](请上传截图1：首页四大入口 - 画一画/说一说/写一写/拍一拍)

2. **3D 精灵 + AI 对话陪伴**
   - 首页有一只用 Three.js 渲染的 3D GLB 模型精灵（猫/狗/萝卜/云朵/团子等多种形象），点击会切换表情、有光晕反馈
   - 首页底部对话框直接跟团子聊天，AI 会引用你原话里的关键词回应（比如你说"论文"，它回信里就有"论文"），不说教、不套话
   - 团子有 4 种预设人设（温暖/理性/元气/安静），还支持自定义性格设定

   ![3D精灵与AI对话](请上传截图2：首页3D精灵 + 对话框 + AI回复)

3. **AI 回信 + 情绪小报**
   - 每次记录后，团子会根据你写的具体内容生成一封专属回信（不是模板，是真的围绕你写的事回应）
   - 周/月会生成情绪小报，统计情绪占比、提取高频关键词、生成何炅风格的温柔总结信

   ![AI回信与情绪小报](请上传截图3：写一写的AI回信 或 情绪小报页面)

## 2. Demo 创作思路

**灵感来源：**
我自己就是个容易 emo 的人。有时候半夜写论文写不动了、面试被拒了、跟朋友闹别扭了，想找人说说，但又觉得"这点小事不至于打扰别人"，于是就憋着。憋久了发现情绪会越积越重。

后来我发现，我需要的不是一个"解决问题"的导师，而是一个"听见我"的存在——它不用给我建议，只要说一句"我知道你今天很辛苦"，就够了。于是我想到做一个情绪陪伴精灵，它永远在线、永远不会嫌烦、永远站在你这边。

**想解决的痛点：**
1. **倾诉门槛高**：跟人倾诉怕被说矫情、怕成为负担、怕对方不懂
2. **情绪记录太工具化**：市面上的情绪 App 大多是"打分+标签"，冷冰冰的，记完更孤独
3. **AI 对话太说教**：很多 AI 陪聊动不动就"你应该""建议你"，反而让人更烦

**为什么做这个方向：**
情绪陪伴是个"慢价值"赛道——不追求即时效率，而是追求"被看见"的感觉。我判断这个方向适合用 AI 做，因为：
- LLM 能真正理解上下文语义（比关键词匹配精准太多）
- 3D 形象能提供拟人化的陪伴感（比纯文字对话温暖）
- 笔触/语音/文字多模态输入，覆盖不同表达习惯的人

## 4. TRAE 实践过程

整个 Demo 从创意到落地，全程在 TRAE IDE 里用 AI 协作完成。下面按板块讲讲每个部分怎么做出来的，以及踩过的坑。

### 4.1 精灵形象的诞生：AI 生图 → 2D 捏脸 → 3D 模型

这是整个项目最有"创造力"的部分，也最能体现 TRAE + AI 工具链的配合。

**第一步：用豆包生成精灵概念图**

我没有美术基础，画不出可爱的精灵形象。于是我先用豆包（或其他 AI 生图工具）生成精灵的概念设计图。我会给豆包描述："一只圆圆的、软软的、像糯米团子一样的精灵，Q 版风格，白色背景，正面朝向，适合做 3D 模型的参考图"。豆包帮我生成了团子、小猫、小狗、小萝卜、小云朵等好几个精灵的 2D 概念图。

![精灵概念图](请上传截图4：豆包生成的精灵概念图 或 character/ 目录下的预览图)

**第二步：把 2D 概念图转成 3D GLB 模型**

有了 2D 概念图后，我用 AI 3D 生成工具（如 Tripo / Rodin / 混元 3D）把概念图转成立体的 GLB 模型。生成时会带贴图和动画，可以直接用 Three.js 的 GLTFLoader 加载。每个精灵都有一套 .glb 模型文件 + .jpg 预览图。

**第三步：用 Three.js 把 3D 模型渲染到网页上**

这一步是 TRAE 帮我写代码的核心。我告诉 TRAE："用 Three.js 加载 GLB 模型，要支持点击切换表情、有光晕、不要自动旋转"。TRAE 生成了 `sprite-3d.js` 模块，关键点：

- 用 `GLTFLoader` 加载 6 种精灵的 GLB 文件（团子/小可/小猫/小狗/小萝卜/小云朵）
- 保留模型自带的材质和贴图，不程序化覆盖颜色，让外观完全呈现 AI 生成的设计
- 三盏灯光：环境光（1.4 强度，保证亮度）+ 主光（1.0）+ 补光（0.6），让模型明亮但不过曝
- 设置 `renderer.outputColorSpace = THREE.SRGBColorSpace`，解决 glTF 颜色偏暗的问题
- 相机 z=3.6，模型缩放 2.0，让精灵在 320×320 的容器里占满画面
- 保留 GLB 自带的 AnimationClip 动画（呼吸/浮动），删除自动旋转（用户反馈"转晕"）
- 点击精灵切换 6 种情绪表情，同时切换情绪光晕颜色（喜悦橙、悲伤蓝、愤怒红、焦虑紫…）

![3D精灵渲染](请上传截图5：TRAE 里调试 Three.js 精灵渲染 或 浏览器里 3D 精灵效果)

### 4.2 捏脸系统：从 React 版移植到纯 JS

这是项目里代码量最大的模块。原本我用 TRAE 做了一个独立的 Next.js + React + TypeScript 捏脸项目（在 `face/` 目录），有完整的 UI 组件库（基于 shadcn/ui）。

**捏脸系统的可调参数非常丰富：**

| 部位 | 可调内容 |
|------|---------|
| 脸型 | round/oval/heart 三种 + 脸宽/脸高滑块 + 8 种肤色 |
| 眼睛 | round/cute/oval/sharp 四种 + 宽/高/间距/垂直位置 + 眼角倾斜 ±30° + 8 种眼色 |
| 眉毛 | natural/arched/straight/angry 四种 + 粗细/高度/长度 + 颜色 |
| 鼻子 | dot/small/round/pointed 四种 + 宽/高 |
| 嘴巴 | smile/grin/neutral/open/cat 五种 + 宽/高 |
| 发型 | 11 种（短发凌乱/侧分/刺猬/波浪/长直/双马尾/丸子头…）+ 12 种发色 |
| 服装 | 12 种颜色 |
| 腮红 | 开关 + 强度 + 6 种颜色 |
| 眼镜 | 无/圆框/方框/墨镜 + 颜色 |

**渲染原理：** 全部用 SVG 绘制，渲染层级从底到顶是：展台 → 身体 → 发型后层 → 耳朵 → 脸型 → 腮红 → 眼睛 → 眉毛 → 鼻子 → 嘴巴 → 发型前层 → 眼镜。这样发型能正确地"包住"脸，眼镜在最上层。

后来为了让主应用不依赖 React 那套重框架，我把 React 版的核心渲染逻辑用 TRAE 移植成了纯 JavaScript 版本（`avatar-render.js`），导出 `renderAvatarSVG(state)` 函数返回完整 SVG 字符串，直接插入 DOM。这样捏脸功能在主应用里也能用，且打包体积小很多。

### 4.3 画一画：Canvas 笔触 + 情绪推断

画一画模块的难点不是"画"，而是"从画里推断情绪"。

**画布实现：** 用原生 Canvas + PointerEvent，支持三种笔刷样式：
- `standard` 标准：干净的实心笔画
- `material` 材质：主线 + 平行偏移纹理线，模拟木纹/金属/布料
- `pattern` 花纹：沿笔画路径每隔 18px 戳印一个装饰图案

**笔触情绪推断：** 每次画一笔，都实时采集这些数据：
- 平均速度（像素/毫秒）
- 平均压力（PointerEvent 的 pressure，触屏才有，鼠标默认 0.5）
- 速度方差（反映稳定性）
- 总笔画数
- 绘画时长

然后按规则推断：高速 + 高方差 + 多笔画 → 焦虑/愤怒；低速 + 轻压力 → 悲伤/无力；中速 + 稳定用力 → 喜悦；慢但用力 → 压抑的愤怒。这个规则虽然简单，但配合三种笔刷的视觉差异，体验上挺有代入感。

### 4.4 说一说：Web Audio + Web Speech 双引擎

说一说模块同时用了两个浏览器原生 API：

**Web Audio API（声学特征采集）：**
- 用 `AudioContext + AnalyserNode` 实时采集麦克风音量
- 32 条波形条实时可视化，音量越大条越高
- 采集 avgVolume（平均音量）、volumeVariance（音量方差）、pauses（停顿次数，音量低于阈值时计数）

**Web Speech API（语音转文字）：**
- `webkitSpeechRecognition`，设置 `lang='zh-CN'`、`continuous=true`、`interimResults=true`
- 实时显示转写文字，录音结束自动重启识别
- 不支持的浏览器会优雅降级（只采集声学，不转文字）

**情绪推断：** 先用文本分析得到基础情绪，再用声学特征修正——高音量 + 高方差 → 愤怒/激动；低音量 + 多停顿 → 悲伤；快语速 → 焦虑；慢语速 + 低音量 → 无力。

### 4.5 写一写：AI 精准情绪分析（项目最核心的 AI 能力）

这是用户最看重的板块，也是我从"本地关键词"升级到"AI 大模型"的过程。

**第一版：本地关键词引擎（`analyzeText`）**
- 建了 6 种情绪的关键词表（joy: 开心/高兴/快乐…；sadness: 难过/伤心/哭…）
- 加了否定词处理：往前看 1-3 个字，如果"我不开心"→ 翻转为 sadness
- 加了标点推断：多个感叹号 → anger；多个省略号 → sadness；多个问号 → anxiety

**问题：** 关键词引擎太粗糙。"今天面试又挂了，我真的好努力"会因为命中"努力"被误判。"真好呢，又加班"反讽完全识别不了。

**第二版：AI 精准情绪分析（`analyzeTextAI`）**
- 调用大模型（方舟 ark-code-latest），给一个严格的 system prompt
- 限定输出 6 种情绪之一的 JSON：`{"emotion":"anxiety","intensity":0.8,"reason":"用户提到面试失败和准备很久，明显焦虑"}`
- 提示 AI 处理反讽、否定句、混合情绪
- `temperature=0.3` 保证输出稳定
- 加了 JSON 提取容错：能处理被 markdown 代码块包裹、前后带解释文字等情况
- 失败时 fallback 到本地关键词分析，保证可用

**实测：** 输入"今天面试又失败了，已经准备了好久，感觉好难过好焦虑"，AI 准确识别为「焦虑」，并给出理由"用户提到面试失败和准备很久"。

![写一写AI情绪分析](请上传截图7：写一写页面 AI 识别情绪 + 生成回信)

### 4.6 AI 回信生成：让团子真的"听见"你

情绪识别只是第一步，更重要的是团子要围绕用户写的具体内容回信。

**Prompt 工程：** 这是反复调试最多的地方。system prompt 里有几条硬规则：
1. "必须引用主人原话里的具体词"——用户说"论文"，回信里必须有"论文"
2. "不准说教，不准用'你应该/建议你/你需要'这种句式"——堵住 AI 最容易犯的毛病
3. "共情优先，先接住感受再说话"——先承认情绪，再回应事件
4. "直接输出回信内容，不加引号、不加解释、不写'团子：'之类的称呼"——保证输出干净

**人设系统：** 4 种预设人设各有不同的 prompt：
- warm（温暖）：像何炅，承认感受，不急着给建议
- rational（理性）：先承认感受，再帮你看清局势
- energetic（元气）：热情积极，但不是虚假正能量
- quiet（安静）：话少，主要说"我在"

还支持自定义性格设定——用户可以写一段性格描述，团子的所有回复都会围绕这个性格展开。

**踩坑：** 一开始没限制 `max_tokens`，模型生成太长导致请求超时。加上 `max_tokens=200` 后响应快了很多，同时保留了模板兜底——AI 失败时不会卡死，会用本地预设文案回复。

### 4.7 首页 AI 对话：带上下文的聊天

首页底部有个对话框，可以直接跟团子聊天（不用写日记）。跟写一写不同的是，对话会带上最近 5 条记录作为上下文，让团子"记得"你最近的状态。

比如你昨天写了"论文被导师打回"，今天在首页说"又改了一版"，团子能接上"论文改了第几版了？别太逼自己"。这种连续性让陪伴感更强。

### 4.8 情绪地图 + 记忆瓶

用 Leaflet + 高德地图瓦片做了一个"情绪地图"：
- 每条带位置的记录会在地图上放一个情绪瓶子标记（颜色对应情绪，emoji 标在瓶子上）
- 按时间顺序连线，形成"情绪轨迹"——你能看到自己这一周在城市的哪些角落开心过、难过过
- 新建记录时可以选位置，支持定位当前位置

### 4.9 手账贴纸系统

给时间线记录加了贴纸功能（手账风）：
- 贴纸库：用户上传任意图片当贴纸，存 localStorage
- HTML5 Drag API 从贴纸抽屉拖到记录上
- 放置后可以再调整位置（自定义 mousedown/mousemove/mouseup）
- 按记录 ID 分组持久化，每条记录的贴纸独立保存

### 4.10 情绪小报：何炅风格回信

周/月会生成情绪小报，包含：
- 情绪占比统计（哪个情绪最多）
- 高频关键词提取（从所有记录文本里抽词，过滤停用词）
- 头条标题（按主导情绪生成，如"这周的光，比想象的多"）
- 何炅风格回信——用 LLM 生成，prompt 强调"用细节、不说教、接纳不完美、有金句"

### 4.11 自定义房间背景 + 时间场景

- **自定义背景：** 设置页上传图片，压缩到 1600px 存 localStorage（最多 6 张），首页背景图可切换
- **时间场景：** 根据当前时间切换白天/黄昏/夜晚氛围，夜晚会在背景生成 40 颗闪烁的星星
- **情绪光晕：** 精灵周围的光晕颜色会跟着当前情绪变（喜悦橙、悲伤蓝、焦虑紫…）

### 4.12 AI 对话集成的踩坑记录（重点讲讲）

这是整个项目最难的部分，踩了一连串的坑：

**坑 1：API Key 端点搞错**
我拿到的是方舟 Agent Plan 个人版的 API Key。一开始按普通 API 文档调 `/api/v3/chat/completions`，返回 401。TRAE 帮我查文档才发现，Agent Plan 要用 `/api/plan/v3/chat/completions`，端点完全不同。

**坑 2：CORS 跨域**
端点改对后，浏览器直连又报 `TypeError: Failed to fetch`——Agent Plan 端点不支持 CORS。解决方案是写同域代理：
- 本地：`proxy.py`（Python http.server，同时服务静态文件 + 转发 /api/chat）
- 线上：Vercel Serverless Function（`/api/chat.js`），把请求转发到方舟
- API Key 存服务端环境变量，前端完全不暴露 key

**坑 3：模型回复不精准**
换用 `ark-code-latest`（Auto 模式自动路由到 doubao-seed）后，回复质量明显提升——测试"论文答辩"场景，AI 真的能提到"导师""准备""紧张"这些关键词，不再是套话。

**坑 4：proxy.py 编码报错**
Windows 上 Python 默认 GBK 编码，打印中文（比如 ✓ 符号）会 `UnicodeEncodeError`。开头加了 `sys.stdout.reconfigure(encoding='utf-8')`，把特殊符号换成 ASCII 才解决。

**坑 5：回信生成超时**
写一写提交后 AI 回信一直转圈。排查发现是 `generateReply` 没限制 `max_tokens`，模型生成太长导致超时。加上 `max_tokens: 200` 后响应快了很多，同时保留模板兜底。

![AI对话调试](请上传截图8：TRAE 里调试 /api/chat 代理 或 浏览器控制台 AI 成功响应)

### 4.13 数据存储与架构

整个应用不需要后端数据库，全部用 localStorage 持久化：
- `store.js` 统一管理状态（精灵配置/人设/记录列表/当前背景/设置）
- 贴纸系统独立存储（`STICKER_KEY` + `PLACED_KEY`），避免侵入主数据
- 自定义背景图压缩后存 localStorage（最多 6 张，防止爆容量）
- 前端模块化（ES Modules），每个功能一个 .js 文件，`app.js` 做入口串联

### 技术栈小结

| 层 | 技术 |
|---|---|
| 前端 | 原生 HTML/CSS/JS（ES Modules 模块化） |
| 3D 渲染 | Three.js + GLTFLoader（加载 AI 生成的 GLB 模型） |
| 捏脸系统 | 纯 SVG 渲染（从 React/TS 版移植），9 大部位 30+ 可调参数 |
| AI 生图 | 豆包生成精灵概念图 |
| AI 3D | 2D 概念图转 GLB 3D 模型 |
| AI 大模型 | 火山引擎方舟 Agent Plan（ark-code-latest，Auto 路由） |
| 语音 | Web Audio API + Web Speech API |
| 地图 | Leaflet + 高德地图瓦片 |
| 画布 | 原生 Canvas + PointerEvent |
| 本地代理 | Python http.server（proxy.py） |
| 线上部署 | Vercel Serverless Function（/api/chat.js） |
| 数据存储 | localStorage（本地持久化，无需后端数据库） |
